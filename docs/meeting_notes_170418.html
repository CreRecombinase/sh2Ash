Meeting notes - 4/18

Jean Morrison

April 18, 2017

  • We briefly reviewed simulation results from two weeks ago. I talked about some of the oracle experiments that I did that seem to indicate that having a bad estimate of \(g_{21}\) is responsible for most of the inflation of type 1 error on the “co-regulation” null case.

  • We moved on to talking about data analysis:
  • Matthew asked if there could be overlap between the Height and LDL GWAS samples which hadn’t occured to me. This could potentially explain some of the correlation we see in effect sizes (follow-up: both studies include BLSA, Family Heart Study, InCHANTI, and LOLIPOP which together make up about 7k individuals which is about 7% of the LDL study and 3% of the height study).

  • There is also a question about if population structure could effect these results — e.g. height is variable across Europe, is LDL also variable?

  • Matthew suggested that we try to compare the likelihood between models going in the two directions. Jean will do some work to try to figure out if the likelihoods are comparable. In general, we would like to formalize some of the logic of model comparison —- i.e. that if we get a significant effect in both directions it may be due to confounding. If we see a much stronger effect only in one direction we are more inclined to believe that direction than the opposite.

  • Currently, my results with real data suggest that we are not learning anything we couldn’t learn using the simple method of ascertaining SNPs based on being significant for one trait and then calculating the correlation in effect size estimates within the selected set of SNPS. Ideally we would like a method that does something different and hoepfully more useful than this. There is a possiblitiy that we only see such strong concordance because the sample sizes are large and that our method is more useful for GWAS with less power (lower sample size). Jean will look into how the method performs using only weak effects.

  • Jean will run some simulations artificially adding in one or two strong eQTLs that are also strong GWAS hits to see if this causes our method to generate false positives in the same way it effects the original Sherlock method.
